## RC1 verdict

```txt
V1.3 keep-alive is ready as a stable opt-in Windows build flavor.
Default behavior remains close-after-response.
This is a connection-reuse win, not a concurrency win.
```

## Do not promote to default yet

Default behavior should remain close-after-response until stronger close-policy tests exist for malformed requests, client disconnects, and longer keep-alive sequences.
