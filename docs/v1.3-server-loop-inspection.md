# V1.3 SERVER LOOP INSPECTION

Branch: `feat/v1.3-keepalive`

## Current Windows control flow

The current Windows assembly server has one accepted socket per handled request.

```txt
mainCRTStartup
  -> accept loop
     -> accept
     -> current_client = accepted socket
     -> handle_client
     -> accept loop
```

Relevant shape:

```asm
.accept_loop:
    call accept
    cmp rax, INVALID_SOCKET
    je .accept_loop

    mov qword ptr [rip + current_client], rax
    call handle_client
    jmp .accept_loop
```

Inside `handle_client`, the server performs one `recv`, parses one request, sends one response, then all paths converge on `.close_client`.

```asm
handle_client:
    call recv
    cmp eax, 0
    jle .close_client
    ...
    call send_response
    jmp .close_client

.close_client:
    mov rcx, qword ptr [rip + current_client]
    call closesocket
    leave
    ret
```

## Why the probe result is expected

`make probe-keepalive` produced:

```txt
probe-keepalive: close-after-response confirmed path=/health first_bytes=112
```

That matches the current control flow. The server closes the accepted socket after the first response, so a second request on the same TCP connection cannot succeed.

## V1.3 target shape

The experimental keep-alive path should split the current behavior into two layers:

```txt
handle_client
  -> request loop
     -> recv one request
     -> parse one request
     -> send one response
     -> if keep-alive allowed, continue
     -> otherwise close
```

Target shape:

```txt
accept -> current_client = socket
       -> handle_client
            request_loop:
              recv
              parse
              send_response
              maybe repeat
            close socket
       -> accept loop
```

## First implementation constraint

Do not change all response headers first. The V1.2 response still advertises `Connection: close`. The first safe implementation pass can still keep that default and only prove the internal loop structure behind an experimental generated build.

Only after the internal loop works should the response header switch from:

```txt
Connection: close
```

to:

```txt
Connection: keep-alive
```

## Guardrails

Keep these checks green while experimenting:

```powershell
make verify
make verify-quiet
make probe-keepalive
```

When the experimental keep-alive path is introduced, add a separate target rather than changing the default binary immediately.
